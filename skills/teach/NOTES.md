# Teaching Notes

## User Profile
- Experienced builder (bigpowers, other projects) — expect to move fast
- Wants authenticity over polish — real artifacts, not marketing copy
- Daily update cadence is hard requirement — architecture must be low-friction
- GitHub wiki + OKF specs will be the content source
- Must align with bigpowers principles and conventions

## Key Constraints
- Site launches **before** development starts
- Pulls from live project artifacts (OKF, bigpowers tools, GitHub)
- Solo dev building it alongside the main project
- No complex backend — keep it simple and git-backed

## Teaching Focus
- NOT: web dev basics (user already has skills)
- YES: architecture patterns for living project sites
- YES: how to structure content so it updates automatically from artifacts
- YES: narrative design for transparent project tracking
