# Mission: See bigspec Whole Before Building It

## Why

bigspec is a greenfield rebuild of **bigpowers** — a spec-driven agentic methodology for solo developers. Having built bigpowers over 41 days (98 releases, 74 skills), Daniel now knows what's load-bearing and what was accidental complexity. He wants a **visual product showcase** — an HTML site that lets him *see the whole project before writing a line of it*: the founding ideas, the building blocks, how they concatenate, the build order, and the product at the end. Being very visual, seeing the system whole is how he'll build it in the right order without repeating bigpowers' mistakes. The site doubles as a public "watch it being built" page during construction.

## Success looks like

- A beautiful, product-showcase-quality HTML page that tells bigspec's story: origin (from bigpowers) → foundational ideas → building blocks → how they connect → build roadmap → final product
- Daniel can look at it and *understand the whole shape of bigspec* — what to build first, second, last
- It makes the "lessons learned from bigpowers" explicit (what to keep, cut, replace) so the rebuild avoids the same errors
- Visitors (other solo devs) can see the vision and follow the build
- Content is grounded in the real artifacts: constitution.md (B0–B10), docs/architecture.md, the 5 movements, the roadmap M0→v1.0→M5

## Constraints

- Very visual learner — diagrams, stacks, chains, timelines over walls of text
- Content must be true to the actual design docs, not invented
- Product-showcase quality (this represents the project publicly)
- Launches before development starts — it IS the pre-build blueprint

## Out of scope

- General web dev fundamentals (Daniel has built bigpowers and other projects — he's a pro)
- Daily auto-updating pipeline / CMS complexity (that was a misread; this is a hand-crafted showcase first)
- Marketing/SEO
